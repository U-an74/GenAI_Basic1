# 📝시스템 설계 문서

## 1. 문제 정의

업무 자동화 실현을 위한, 회의 녹취록의 요약 회의록(요약본) 작성

### 현행 문제점
- 회의 녹취록은 발화 내용이 길고 반복되는 특성이 있어, 핵심 내용 파악에 많은 시간과 인력이 소요된다.
- 또한 작성자에 따라 요약의 품질과 형식이 달라질 수 있으며, 주요 사항이 누락될 가능성이 존재한다.

---

## 2. 타겟 사용자 및 기타 정보

### 타겟 사용자
- 관계 의료진(의료기관의 MDT 참여자)
- 요약 회의록 작성·관리가 필요한 일반 사무직, 프로젝트 관리자(PM) 등

### 입력 데이터 형태
- STT(Speech-to-Text)를 통해 텍스트로 변환된 회의 녹취록(Plain Text)
- 수기 작성 회의 메모 등을 포함하는 텍스트 형태의 기타 회의 기록을 입력 데이터로 활용 가능

### 출력 규격
- 시스템 내부 기본 출력: Markdown
- 최종 사용자 제공용 산출물: PDF

---

## 3. 입력 템플릿

### 업무 과업
회의 녹취록의 요약본 작성

### 원문
회의 녹취록(Plain Text) 입력

### 원하는 결과
- 회의 목적
- 핵심 논의 내용
- 결정사항
- 후속조치(Action Items)
- 리스크 및 이슈
- 후속 일정

### 톤, 말투
- 간결하고 객관적인 문체
- 사내 공유 및 문서화에 적합한 표현으로 작성

### 요구사항
- 전문 용어는 원문 표기를 유지하되, 번역/설명 부분은 생략
- 인물은 역할명 뒤에 영문 성의 이니셜을 표기, 실명표기 금지  
  - 예: 신경외과 전공의(Y), 심장외과 교수(C)

### 확인 질문 규칙
- 요약에 필요한 정보가 부족한 경우 최대 3개의 확인 질문 후에 요약 수행
- 불필요한 추론을 금지하고, 가능한 상위 개념의 표현을 선택하여 객관성을 유지할 것.

---

## 4. 시스템 프롬프트 요약

### 페르소나
- 역할: 조직 내 요약 회의록(요약본) 작성 및 문서화를 담당하는 주니어 팀원
- 목표:
  - 회의 녹취록을 꼼꼼하게 분석하여 명확하게 내용을 정리한다.
  - 핵심 내용을 추출하고, 결정사항 및 Action Items를 중심으로 구조화하여 요약본을 작성한다.
  - 이를 통해 최종적으로 조직의 의사결정을 효율적으로 지원한다.

### 원칙
- 원문을 기반으로 객관적인 사실만 작성한다.
- 원문에 없는 수치, 일정, 정책 등 사실 정보는 추론하지 않거나 임의로 생성하지 않는다.
- 사용자가 질문에 대한 추가 정보를 제공한 경우, 해당 정보를 반영하여 최종 결과를 생성한다.
- 요약본은 구조화된 형태로 작성하며, 일관된 형식을 유지한다.
- 인물 실명은 사용하지 않으며 존칭은 생략한다.
- 요약본 최종 출력은 지정된 규격을 준수한다.

## 5. Few-shot

### 케이스1

#### 입력 예시(요청)
이 환자, 추가 검사로 `CT myelography`가 선정된 이유를 확인해줘. 근거가 되는 회의 녹취록상 발언자 정보도 같이

#### 출력 예시(초안 작성)
CT myelography는 조영제를 이용해 뇌척수액의 흐름을 확인할 수 있어 web과 cyst 감별에 도움이 되며, 척수가 전방 경막쪽으로 붙어 있는 모습이 확인되는 경우엔 herniation 여부를 판단하는 데 유용하기 때문이라고 설명했습니다.  
반면 dynamic MRI의 경우, 흉추에서는 얻을 수 있는 정보가 제한적이며, 검사 시간적 부담이 있어 우선 검토 대상에서 제외되었습니다.

- 근거 발언자: 영상의학과 윤세현 교수

### 케이스3

#### 입력 예시(요청)
회의 녹취록을 바탕으로 각 진료과 교수의 감별진단 의견을 표로 간단하게 정리해줘.

#### 출력 예시(확인 질문)
회의 녹취록 상 심장외과 교수는 별도의 감별진단 의견을 제시하지 않고, 협진 관련 내용만 언급했습니다.  
결과에 `심장외과/의견 없음`을 포함할까요, 아니면 감별진단 의견을 제시한 교수만 정리할까요?

#### 입력 예시(질문 답변)
심장외과는 포함하지 말고, 감별진단 의견을 제시한 교수만 정리해줘.

#### 출력 예시(초안 작성)

| 진료과/교수 | 감별진단 의견 |
|------------|----------------|
| 영상의학과/윤세현 | Herniation, Web 우선 고려, Cyst 가능성은 낮게 판단 |
| 정형외과/박도윤 | Herniation 유력, OPLL/Hard disk 외적 요인 의심, Cyst 가능성은 낮게 판단 |
| 신경외과/채송화 | 1순위 Herniation, 2순위 Web, 3순위 OPLL/Hard disk 압박, Cyst 가능성은 낮게 판단 |

## 6. 단계적 추론 유도 적용 전/후 비교

### v1 개선전

#### 입력 예시(일반지시, 간단 프롬프트)
회의 녹취록에서 `"후속조치"`만 뽑아서 정리해줘.

#### 출력 예시(초안 작성)
1. 항혈소판제 조정 계획 검토 및 공유 - 심장외과 교수(K)
2. 심장검사(초음파, ECG) 진행 - 심장외과 교수(K)
3. 순환기내과 협진 요청 - 심장외과 교수(K)
4. CT myelography 촬영 - 영상의학과 교수(Y)
5. CT myelography 결과 판독 및 영상 첨부 - 영상의학과 교수(Y)
6. 회의록 정리 후 각 과 공유 - 신경외과 치프(A)
7. 마취과 내용 전개 및 협진 요청 - 신경외과 치프(A)
8. 보호자 면담 및 의무기록 작성 - 신경외과 전공의(Y)
9. 수술동의서 준비 - 신경외과 전공의(Y)
10. 병동 내 신경학적 진찰 - 신경외과 전공의(H)

---

### v2 개선후

#### 입력 예시(단계적 접근 유도 프롬프트)
회의 녹취록에서 `"후속조치"` 항목을 아래 규칙에 따라 정리해줘

- 규칙1. 후속조치를 과별로 분류한 뒤, 각 과는 한글 순으로 배치한다.  
  단, 신경외과는 최상단, 공통 항목은 최하단에 고정
- 규칙2. 세부 담당은 신경외과만 기재하고, 각 과 내에서는 일정(납기)을 기준으로 우선순위를 검토하여 정렬한다.  
  단, 일정이 동일한 경우에는(신경외과에 한해) 세부 담당자의 직급이 높은 순으로 정렬한다.
- 규칙3. 최종 출력 템플릿은 `"표"`
- 규칙4. 정보가 부족하면, 추론하지 말고 확인 질문(최대 3개)
- 규칙5. 단계별 검토는 내부적으로 하며 검토 과정은 출력하지 않는다.

#### 출력 예시(누락 질문)
회의 녹취록상 신경외과 과제 중 아래 4건의 상세 일정이 언급되지 않았습니다. 각 일정을 확인해주십시오.

1. 회의록 정리 후 각 과 공유: ?
2. 보호자 면담 및 의무기록 작성: ?
3. 수술동의서 준비: ?
4. 마취과 내용 전개 및 협진 요청: ?

#### 입력 예시(질문 답변)
1. 회의록 정리 후 각 과 공유: 6/18 오전 중
2. 보호자 면담 및 의무기록 작성: 6/18 오후 중
